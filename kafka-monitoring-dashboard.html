üö® Alert Rules (Copy to Prometheus)

groups:
- name: kafka_offset_commit_issues
  interval: 30s
  rules:
  
  # CRITICAL: Offsets not being committed
  - alert: OffsetsNotCommitted
    expr: |
      (kafka_consumer_last_processed_offset - on(topic,partition,groupId) kafka_consumer_last_stored_offset) > 1000
    for: 5m
    labels:
      severity: critical
      team: platform
    annotations:
      summary: "Kafka offsets not being committed for partition {{ $labels.partition }}"
      description: "{{ $value }} messages processed but not committed. Background tasks likely stuck."
      runbook: "Check consumer_oldest_background_task_age_ms. Consider restart if > 5 minutes."
  
  # WARNING: No offset progression
  - alert: NoOffsetProgression
    expr: |
      rate(kafka_consumer_last_stored_offset[5m]) == 0 
      AND rate(kafka_consumer_last_processed_offset[5m]) > 0
    for: 10m
    labels:
      severity: warning
    annotations:
      summary: "Offsets stuck for partition {{ $labels.partition }} despite processing"
      description: "Consumer processing but not committing. Check background tasks."
  
  # INFO: High offset store failure rate
  - alert: HighOffsetStoreFailureRate
    expr: |
      sum(rate(kafka_consumer_offset_store_attempts_total{status="failed"}[5m])) by (partition)
      / sum(rate(kafka_consumer_offset_store_attempts_total[5m])) by (partition) > 0.1
    for: 5m
    labels:
      severity: info
    annotations:
      summary: "High offset store failure rate for partition {{ $labels.partition }}"
      description: "{{ $value | humanizePercentage }} of offset stores failing"

‚úÖ Quick Diagnosis Steps

  1. Check offset delay: If last_processed - last_stored > 1000 ‚Üí Problem confirmed
  2. Verify processing: If rate(last_processed_offset) > 0 ‚Üí Consumer is alive
  3. Check commits: If rate(last_stored_offset) = 0 ‚Üí Offsets not committing
  4. Confirm pattern: If delay drops to 0 after restart ‚Üí Background task issue confirmed
Solution: The metrics will help identify which background tasks are stuck. Look for tasks with consumer_oldest_background_task_age_ms > 300000 (5 minutes)

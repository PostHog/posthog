# Tab awareness tracker

Which scenes are tab aware (their `SceneExport` root logic is keyed by the internal tab id, so every open tab holds its own instance and state) and which aren't.
A scene becomes tab aware by adding `tabAwareScene()` (or a `key` that includes `props.tabId`) to its root logic and threading `tabId` into child logics — see the `making-scenes-tab-aware` skill for the conversion playbook.
Regenerate the lists with the audit commands in that skill's references.

## Known cross-cutting gaps

- Child logics that reference a tab-aware scene root **without** a `tabId` fall back to a shared `__no_tab__` instance (see `tabAwareScene`) — they work, but do not see the per-tab instance the scene renders. Grep for the console warning `built without a tabId prop` at runtime to find offenders; known ones: `customer_analytics` account children, `engineering_analytics` data logic.
- `products/posthog_ai` visualization widget lost its per-tab insight-suggestion wiring in the tool-widget rewrite.
- Restored desktop tabs adopt the persisted tab set after the first scene loads, so the first scene re-keys once at startup.

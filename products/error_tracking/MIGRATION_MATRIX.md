# Error tracking external import migration matrix

This checklist turns the high-level isolation plan in `PLAN.md` into a file-by-file backlog.
It covers every current external import site outside `products/error_tracking/` that still reaches into Error tracking internals.

Legend:

- `P0` — migrate next; business-boundary leak with a clear facade target
- `P1` — migrate after P0; still avoidable, but needs a slightly larger API addition
- `P2` — migrate when the product-facing API exists; mostly tooling/tests
- `P3` — do not force through the business facade; move to an explicit public infra/framework boundary instead
- `keep` — intentional framework registration path; no migration needed for facade isolation

## Proposed new contracts and APIs

The current facade already covers basic issue reads, counts, assignments, remote config, and weekly-digest helpers.
The remaining gaps are small and should stay capability-oriented.

### Existing contracts to reuse

- `IssueSummary`
- `ErrorTrackingIssueContract`
- `ErrorTrackingIssueAssignmentContract`
- `TeamCountContract`

### Proposed contracts to add next

```python
@dataclass(frozen=True)
class IssueFingerprintContract:
    issue_id: str
    team_id: int
    fingerprint: str
    version: int
    first_seen: datetime | None
    created_at: datetime

@dataclass(frozen=True)
class IssueListItemContract:
    id: str
    team_id: int
    name: str | None
    description: str | None
    status: str
    first_seen: datetime | None
    last_seen: datetime | None
    volume: int | None
    first_event: dict[str, Any] | None

@dataclass(frozen=True)
class ErrorTrackingIssueQueryResultContract:
    results: list[IssueListItemContract]

@dataclass(frozen=True)
class IssueFingerprintOverrideInput:
    team_id: int
    issue_id: str
    fingerprint: str
    version: int

@dataclass(frozen=True)
class DeleteSummaryContract:
    deleted_count: int
```

### Proposed facade / public-boundary additions

- `delete_issue_fingerprints_for_teams(team_ids: list[int]) -> DeleteSummaryContract`
- `iterate_issue_fingerprints(team_id: int, start: datetime, end: datetime) -> Iterable[IssueFingerprintContract]`
- `find_issue_fingerprint(team_id: int, fingerprint: str) -> IssueFingerprintContract | None`
- `override_issue_fingerprint(input: IssueFingerprintOverrideInput) -> None`
- `query_issues(team: Team | int, query: ErrorTrackingQuery) -> ErrorTrackingIssueQueryResultContract`
- `public_tools.SearchErrorTrackingIssuesTool` or `max_tools.SearchErrorTrackingIssuesTool`
- `backend.infra.schema.*` and `backend.infra.embedding.*` for intentional non-facade public constants

## Recommended follow-up PR order

1. **PR 1 — test-only boundary cleanup**
   - Replace cross-product ORM imports in external tests with sanctioned Error tracking test factories.
2. **PR 2 — admin/cleanup facade**
   - Add fingerprint iteration, deletion, and override APIs for management commands and team cleanup.
3. **PR 3 — query facade**
   - Add a typed facade entry point for the main issues query and migrate Signals + example script.
4. **PR 4 — framework adapter split**
   - Move `posthog/hogql_queries/query_runner.py` imports off internal product runner modules and into an explicit framework adapter boundary.
5. **PR 5 — public tool entry point**
   - Stop importing AI tools from product-internal module paths.
6. **PR 6 — infra module extraction**
   - Extract `backend.sql`, `backend.embedding`, and `backend.indexed_embedding` into a clearly public `backend.infra.*` surface and update `tach.toml`.

## Exit criteria for this matrix

This migration backlog is complete when:

- all avoidable cross-product callers import only `products.error_tracking.backend.facade` or sanctioned test/public helper paths,
- framework wiring imports only `backend.presentation.*`, `backend.infra.*`, or DAG/app registration paths,
- `tach.toml` reflects those public boundaries explicitly,
- no external caller imports `backend.models`, `backend.hogql_queries.*`, `backend.tools.*`, `backend.sql`, `backend.embedding`, or `backend.indexed_embedding` by accident.

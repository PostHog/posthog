How this is computed

Orgs are classified into one cohort (Enterprise > Scale > Boost > Paid > Free) from their synced available_product_features. Add-ons stack, so each feature is shown for every cohort that includes it. Denominator = cohort size; usage = the backing Postgres model (no events).

Caveats: billing overrides can shift an org ±a tier; the deprecated Teams add-on overlaps Boost/Scale and folds into them. Authoritative add-on membership lives in the billing service.
